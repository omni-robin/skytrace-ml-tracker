# 2026-03-15

## SkytraceRT_poc
- Continued work on improving postprocess toward tighter edges and unmerging, but concluded the occupancy→bands approach is misaligned with the hard requirement: **no merged controllers** and **edge tolerance ±0.4% of GT bandwidth**.
- Added split-parameter grid support to `SkytraceRT_poc/scripts/sweep_postprocess.py` (plus `--limit`), but long sweeps can be slow / sometimes SIGKILL due to cost/log spam.

## New project: skytrace-ml-tracker (next-gen)
Goal: build an Orin-friendly, retrainable ML+tracking approach that separates multiple LoRa controllers (no merges) and hits ±0.4% BW edge accuracy.

### Repo created
- GitHub repo created + pushed:
  - https://github.com/omni-robin/skytrace-ml-tracker

### Evaluator harness (hard gates)
- Implemented strict evaluator:
  - 1:1 greedy IoU matching (no merges interpreted as `matched_n == gt_n`)
  - edge gate: per matched band
    - |pred_lower-gt_lower|/gt_bw <= 0.004
    - |pred_upper-gt_upper|/gt_bw <= 0.004
- Added unit tests.
- Added a GT-cheat sanity script which confirms evaluator is correct end-to-end:
  - `scripts/eval_gt_cheat.py` → strict_ok=all, all_edges_ok=all.

### Baseline check
- Ran the existing SkytraceRT_poc predictor through the strict evaluator:
  - subset20 limit10: strict_ok=0, all_edges_ok=0 (expected; merges and edges too loose).

### ML v0 proposals implemented (but not passing yet)
- Added ML v0: `TinyCenterBwNet` (1D conv over normalized log-PSD) predicting:
  - per-bin center logits (multi-peak)
  - per-bin bandwidth (log1p bw_bins) trained only at GT center bins
- Scripts added:
  - `scripts/build_train_npz.py` (SigMF → log-PSD features + dense targets)
  - `scripts/train_centerbw.py`
  - `scripts/infer_centerbw.py`
  - `scripts/eval_centerbw_folder.py`
- Quick training on subset20 (tiny dataset ~20 samples) produced proposals but evaluation still:
  - subset20 limit10: strict_ok=0, all_edges_ok=0.
- Conclusion: ML v0 is scaffolding; to satisfy requirements we likely need a spectrogram-based model that can learn chirp slope/structure and/or explicitly predict left/right edges.

### Next step (tomorrow)

- Keep strict evaluator as the gating signal.
- After ML v1 produces separated controller candidates, add gap-tolerant association/tracking for retune scenarios.
